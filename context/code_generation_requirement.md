# Code Requirements

## Architecture

* Follow a modular architecture
* Keep components small and focused
* Separate UI, business logic, and data access
* Organize files by feature when appropriate

## Reusability

* Create reusable components whenever possible
* Avoid duplicate code
* Shared UI elements should be reusable across the application

### Examples

* Modal container should be reusable for all modal dialogs
* Buttons should use shared components
* Cards should use shared components
* Form inputs should use shared components

## Code Quality

* Follow React and TypeScript best practices
* Write clean, readable, and maintainable code
* Use meaningful naming conventions
* Avoid unnecessary complexity

## File Size

* Prefer keeping files under 600 lines
* Split large components into smaller components when needed
* Extract reusable logic into hooks and utilities

## Maintainability

* Keep folder structure organized
* Use consistent coding patterns
* Minimize tight coupling between components
* Design for future scalability

## Performance

* Avoid unnecessary re-renders
* Reuse components and logic where possible
* Keep the application responsive and efficient
